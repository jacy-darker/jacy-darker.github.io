    并查集¶

    基本思想¶

    并查集主要用于解决一些元素分组的问题。它管理一系列不相交的集合，并支持两种操作：

    • 合并Union：把两个不相交的集合合并为一个集合。
    • 查询Find：查询两个元素是否在同一个集合中。查找其父节点

    要素¶

    • 底层数据结构可以根据需要选择为：vector,map等

    • 初始化：初始化每个节点的父节点为自己

    if (rank[x] <= rank[y]) {
        fa[x] = y; // 相同深度时，也指向y
    } else {
        fa[y] = x;

    }                   
}

    问题¶

    在由 1 x 1 方格组成的 n x n 网格 grid 中，每个 1 x 1 方块由 '/'、'\' 或空格构成。这些字符会将方块划分为一些共边的区域。

    思路¶

    将一个单元格划分为4个小区域，根据单元格的字符，对4个区域进行合并。

    此外，不同单元格之间也要进行合并。方法是可以选择向右、向下合并。

int regionsBySlashes(vector<string>& grid) {
    int n = grid.size();
    int N = 4*n*n;
    fa = vector<int>(N,0);
    rank = vector<int>(N,1);
    for(int i=0;i<N;i++) fa[i] = i;
    for(int i=0;i<n;i++){
        for(int j=0;j<n;j++){
            int index = 4*(i*n+j);
            if(grid[i][j]=='/'){
                merge(index,index+1);
                merge(index+2,index+3);
            }
            else if(grid[i][j]=='\\'){
                merge(index,index+3);
                merge(index+1,index+2);
            }
            else{
                merge(index,index+1);
                merge(index,index+2);
                merge(index,index+3);
            }
            if(j+1<n) merge(index+3,4*(i*n+(j+1))+1);
            if(i+1<n) merge(index+2,4*((i+1)*n+j));
        }
    }
    unordered_map<int,int> mp;
    for(int i=0;i<N;i++){
        mp[find(i)] = 1;
    }
    return mp.size();
}

    字典树¶

    基本思想¶

    字典树又叫Trie树、前缀树、单词查找树或键树，是一种哈希树的变种。典型应用是用于统计和排序大量的字符串，所以经常被搜索引擎系统用于文本词频统计。它的优点是：利用字符串的公共前缀来减少查询时间，最大限度地减少无谓的字符串比较。Trie的核心思想是空间换时间。利用字符串的公共前缀来降低查询时间的开销以达到提高效率的目的。

    性质¶

    • 根节点不包含字符，除根节点外每一个节点都只包含一个字符
    • 从根节点到某一节点，路径上经过的字符连接起来，为该节点对应的字符串
    • 每个节点的所有子节点包含的字符都不相同

    要素¶

    • 节点的成员变量

      • 子结点
      • 标志位记录是否是叶子结点
      • 针对问题对应的value（类似于map的value）

    • 插入一个新的字符串

    • 检查一个字符串是否在前缀树中
    • 检查前缀树中是否存在一段前缀

    class Trie {
private:
    //每个节点都有26个子节点，但有的子节点是空值
    vector<Trie *> children;
    //记录是否是叶子节点
    bool isEnd;
    //相当于map的值
    int value;

    Trie *searchPrefix(string prefix) {
        Trie *node = this;
        for (char ch: prefix) {
            ch -= 'a';
            if (node->children[ch] == nullptr) {
                return nullptr;
            }
            node = node->children[ch];
        }
        return node;
    }

public:
    Trie() : children(26), isEnd(false) {}

    //向前缀树中插入字符串 word
    void insert(string word) {
        Trie *node = this;
        for (char ch: word) {
            ch -= 'a';
            if (node->children[ch] == nullptr) {
                node->children[ch] = new Trie();
            }
            node = node->children[ch];
        }
        node->isEnd = true;
    }

    //返回字符串 word 是否在前缀树中
    bool search(string word) {
        Trie *node = this->searchPrefix(word);
        return node != nullptr && node->isEnd;
    }

    //是否存在之前已经插入的字符串 word 的前缀之一为prefix
    bool startsWith(string prefix) {
        return this->searchPrefix(prefix) != nullptr;
    }
};

    线段树¶

    基本思想¶

    线段树是一种二叉搜索树，线段树的每个结点都存储了一个区间，也可以理解成一个线段，而搜索，就是在这些线段上进行搜索操作得到你想要的答案。线段树可以用来对某一段区间的值进行统一更新，求某一段区间的值，比如和、最值等。每次更新或查询操作的时间复杂度都是\(O(log \ n)\)，这也意味着渗透着二分的思想。

    分类¶

    • 节点值分类：节点代表某一段区间，节点值代表区间的一种属性
      • 区间和
      • 区间最值
    • 区间更新
      • 加减
      • 覆盖

    要素¶

    • 节点：一个节点代表一段区间，节点的值代表着该区间的值

    • 节点代表的区间编号隐藏在了它在整个二叉搜索树中的位置中。比如一个序号为i的节点的左节点是2*i，右节点是2*i+1

    • 懒惰标记：当对区间进行更新操作时，不需要更新所有在此区间内的节点值。相反，可以将对某一段区间的更新尽可能少的添加到一些子区间上，将更新值记录在懒惰标记中，而不需要对其更小的叶子节点进行更新。因此，在代码中，当找到一个节点所代表的区间完全在更新区间内的时候，只需要更新它的懒惰标记即可，不需要计算它的子区间的值。对应到查询的时候，当检查到某个一节点没有下放懒惰标记时，需要立马下放懒惰标记，同时将自己的懒惰标记清零。

    • 对应于不同的节点值含义和区间更新方式，懒惰标记和节点值的更新方式也不一样，可以对应于代码理解

    节点值类型 区间更新类型 节点值更新 懒惰标记下推
    区间和 加减 node.val += N(叶子结点) * val node->add += val
    区间和 覆盖 node.val += N(叶子结点) * val node->add = val
    区间最值 加减 node.val += val node->add += val
    区间最值 覆盖 node.val = val node->add = val
    • 动态开点：为了避免不必要的空间消耗，使用动态的方式在需要的时候new一个新的节点。但是在c++中new操作是很费时间的，有时候会成为算法超时的原因。每次使用new的时候它会在分配的那块内存前面添加 32 byte的redzone来检测越界错误，导致整个程序的内存消耗比理论值要大。为此可以选择自定义new操作或使用数组，本质上都是一次性分配大量空间。

class SegmentTree {
public:
    Node *root = new Node();

    //区间/单点操作：对区间[start,end]中[l,r]范围内的区间+val
    void update(Node *node, int start, int end, int l, int r, int val);

    //查询区间：在区间[start,end]查询[l,r]的值
    //查询时需要下推懒惰标记，将懒惰标记清零
    int query(Node *node, int start, int end, int l, int r);

private:
    //更新当前节点的值
    void pushUp(Node *node);

    //下推懒惰标记，leftNum 和 rightNum 表示左右孩子区间的叶子节点数量
    void pushDown(Node *node, int leftNum, int rightNum);
};

void SegmentTree::update(Node *node, int start, int end, int l, int r, int val) {
    //区间[l,r]在[start,end]里面
    if (l <= start && end <= r) {
        node->val += (end - start + 1) * val; // 根据题意看是否要乘
        node->add += val;
        return;
    }
    //区间[l,r]不完全在[start,end]里面
    int mid = (start + end) >> 1;
    //下推标记
    pushDown(node, mid - start + 1, end - mid);
    if (l <= mid) update(node->left, start, mid, l, r, val);
    if (r > mid) update(node->right, mid + 1, end, l, r, val);
    pushUp(node);
}

int SegmentTree::query(Node *node, int start, int end, int l, int r) {
    if (l <= start && end <= r) return node->val;
    int mid = (start + end) >> 1;
    int ans = 0;
    pushDown(node, mid - start + 1, end - mid);
    if (l <= mid) ans += query(node->left, start, mid, l, r);
    if (r > mid) ans += query(node->right, mid + 1, end, l, r);
    return ans;
}

void SegmentTree::pushUp(Node *node) {
    node->val = node->left->val + node->right->val;
}

void SegmentTree::pushDown(Node *node, int leftNum, int rightNum) {
    if (node->left == nullptr) node->left = new Node();
    if (node->right == nullptr) node->right = new Node();
    if (node->add == 0) return;
    node->left->val += node->add * leftNum;
    node->right->val += node->add * rightNum;
    node->left->add += node->add;
    node->right->add += node->add;
    node->add = 0;
}

    int sumCounts(vector<int>& nums) {
    constexpr int mod = 1e9+7;
    long long res = 0;
    long long cur = 0;

    树状数组¶

    基本思想¶

    树状数组是一种能够高效求解区间问题的数据结构，可以实现在\(O(log n)\)的时间复杂度下实现区间修改和区间查询。树状数组基于差分的思想对区间数据进行修改，为了应对差分数组求前缀和的需要\(O(n)\)的复杂度，采用了基于二进制思想的区间划分方法，在保证了数据完整性的同时减少了求前缀和时累加的次数，实现数据的高效读写。

    树状数组适用于区间的求和问题，但不适合区间的最值问题（与线段树相比，不支持区间运算不可逆的问题，如最值问题等），更一般的，树状数组能做的线段树都能做。此外，树状数组构建的时间复杂度是O(nlog n)，而线段树是O(n)。

    数学原理¶

    • 维护成员

      • diff：原始数据的差分数组，只在构建树状数组的时候使用

      • tree：差分数组diff[i]的树状数组

      • helpTree：差分数组i*diff[i]的树状数组

    • 数学原理

    \[ \begin{aligned} sum(l,r) &= preSum(r) - preSum(l-1) \\ \end{aligned} \]
    \[ \begin{aligned} preSum(k) &= n[0] + n[1] + \cdots + n[k] \\ &= d[0] + (d[0]+d[1]) + \cdots + (d[0]+d[1] + \cdots + d[k]) \\ &= (k+1) \times d[0] + k\times d[1] +\cdots + d[k] \\ &= (k+1) \times (d[0] + d[1] + \cdots + d[k])-(0\times d[0]+1\times d[1] + \cdots + k\times d[k]) \\ &= (k+1)\times preSumTree(k) - preSumHelpTree(k) \end{aligned} \]

    至此，区间求和转换成了求两个差分数组的前缀和问题。对于一般数组，求前缀和只需要累加即可，但这样的时间复杂度是\(O(n)\)。实际上，树状数组采用了不同的区间划分方式，虽然仍然分为n个节点，但每个节点代表的子区间的长度不一定是1，这样累加的时候就可以更加高效。

    为了下面我们将两个差分数组使用树状数组来对他们进行表达。可以得出，对树状数组进行单点修改和区间求和的时间复杂度是\(O(log n)\)。而由于他们代表的数据是差分，所以

    1
2
    - 原数组的区间修改`O(n)`$\Longrightarrow$差分数组的单点修改`O(1)`$\Longrightarrow$树状数组的单点修改`O(log n)`
- 原数组的区间求和`O(n)`$\Longrightarrow$差分数组的区间求和`O(n)`$\Longrightarrow$树状数组的区间求和`O(log n)`
    • 树状数组的构建

    首先介绍一下子区间划分，对于一个长度为n的子区间，维护了n个以\(i\in[0,n-1]\)为右边界的子区间所代表的区间和，每个子区间的左边界使用i-lowbit(i)计算，结果是将右边界以二进制表示后，将最低位的1换成0得到。

    • 树状数组的前缀和（原数据区间求和）：

    class BIT {
public:
    vector<int> diff, tree, helpTree;
    int n;

    BIT(vector<int> &nums) {
        this->n = nums.size();
        this->diff = vector<int>(n, 0);
        this->tree = vector<int>(n, 0);
        this->helpTree = vector<int>(n, 0);
        diff[0] = nums[0];
        for (int i = 1; i < n; i++) diff[i] = nums[i] - nums[i - 1];
        for (int i = 0; i < n; i++) {
            add(tree, i, diff[i]);
            add(helpTree, i, i * diff[i]);
        }
    }

    //区间修改: +x
    void update(int l, int r, int x) {
        add(tree, l, x);
        add(tree, r + 1, -x);
        add(helpTree, l, l * x);
        add(helpTree, r + 1, (r + 1) * (-x));
    }

    //单点修改: +x
    void add(vector<int> &thisTree, int k, int x) {
        //对所有包含下标k的区间进行修改
        for (int i = k + 1; i <= n; i += lowbit(i)) {
            thisTree[i - 1] += x;
        }
    }

    //求前缀和
    int preSum(vector<int> &thisTree, int k) {
        int ans = 0;
        for (int i = k + 1; i > 0; i -= lowbit(i)) ans += thisTree[i - 1];
        return ans;
    }

    //区间查询
    int sum(int l, int r) {
        int preSumL = l * preSumL(tree, l - 1) - preSum(helpTree, l - 1);
        int preSumR = (r + 1) * preSum(tree, r) - preSum(helpTree, r);
        return preSumR - preSumL;
    }

    int lowbit(int i) {
        return i & -i;
    }
};

    for (int i = 0; i < n; i++) {
        nums[i] = lower_bound(SortNums.begin(),SortNums.end(),nums[i])-SortNums.begin()+1;
    }
}

    int n = nums.size();
    set<int> sortNums;
    for(int x:nums) sortNums.insert(x);
    unordered_map<int, int> mp;

        for (int x : allNumbers) {

        }

}

    统计逆序对¶

    思想¶

    统计逆序对是树状数组的一个经典用法。对于数组nums[n]

    • 首先将将树状数组的原形数组定义为一个长度为n的零数组，该数组我们称为数组origin，使用树状数组tree对其进行维护
    • 对原数组nums[n]进行离散化，将元素值改为在数组中按顺序出现的位置（第几小元素）
    • 按照逆序的方式遍历原数组
    • 对于元素nums[i]，它按顺序应该出现的位置是nums[i]-1处，将origin[nums[i]-1]元素值+1。那么对于下一次要遍历的元素nums[i+1]，它应该出现在nums[i+1]-1处，如果nums[i+1]-1下标之前存在origin[k]=1，说明元素nums[i+1]之前存在元素比它小。事实上，nums[i+1]-1下标之前数组origin的前缀和就代表了出现了多少次这种情况

class BIT{
public:
    vector<int> tree;
    int n;
    BIT(int n){
        this->n = n;
        this->tree = vector<int>(n,0);
    }
    int preSum(int k){
        int ans = 0;
        for(int i=k+1;i>0;i -= lowbit(i)) ans += tree[i-1];
        return ans;
    }
    void add(int k,int x){
        for(int i=k+1;i<=n;i += lowbit(i))  tree[i-1] += x;
    }
    int lowbit(int i){
        return i & -i;
    }
};

void discrete(vector<int> &nums){
    int n = nums.size();
    vector<int> SortNums = nums;
    sort(SortNums.begin(),SortNums.end());
    for(int i=0;i<n;i++){
        nums[i] = lower_bound(SortNums.begin(),SortNums.end(),nums[i])-SortNums.begin()+1;
    }
}

class Solution {
public:
    int reversePairs(vector<int>& record) {
        int n = record.size();
        int res = 0;
        BIT bit(n);
        discrete(record);
        for(int i=n-1;i>=0;i--){
            res += bit.preSum(record[i]-2);
            bit.add(record[i]-1,1);
        }
        return res;
    }
};

    翻转对¶

    给定一个数组 nums ，如果 i < j 且 nums[i] > 2*nums[j] 我们就将 (i, j) 称作一个*重要翻转对*。

    你需要返回给定数组中的重要翻转对的数量。

    示例 1:

    1
2
    输入: [1,3,2,3,1]
输出: 2

    示例 2:

    1
2
    输入: [2,4,3,5,1]
输出: 3

    注意:

    1. 给定数组的长度不会超过50000。
    2. 输入数组中的所有数字都在32位整数的表示范围内。

    思路¶

    树状数组支持的基本查询为求出\([1,val]\)之间的整数数量。因此，对于元素nums[i]，只需要统计[1,nums[i]*2]的个数，以及[1,maxValue]的个数，两者相减即可。

    由于数组中整数的范围可能很大，故在使用树状数组解法之前，需要利用哈希表将所有可能出现的整数，映射到连续的整数区间内。

    int n = nums.size();

int reversePairs(vector<int>& nums) {
    int res = 0;
    auto mp = discrete(nums);
    int n = nums.size();
    BIT bit(mp.size());
    for(int i=0;i<nums.size();i++){
        int left = mp[(long long)nums[i] * 2];
        int right = mp.size();
        res += bit.preSum(right-1) - bit.preSum(left-1);
        bit.add(mp[nums[i]]-1, 1);

    }
    return res;
}

    统计包含每个点的矩形数目¶

    给你一个二维整数数组 rectangles ，其中 rectangles[i] = [li, hi] 表示第 i 个矩形长为 li 高为 hi 。给你一个二维整数数组 points ，其中 points[j] = [xj, yj] 是坐标为 (xj, yj) 的一个点。

    第 i 个矩形的 左下角 在 (0, 0) 处，右上角 在 (li, hi) 。

    请你返回一个整数数组 count ，长度为 points.length，其中 count[j]是 包含 第 j 个点的矩形数目。

    如果 0 <= xj <= li 且 0 <= yj <= hi ，那么我们说第 i 个矩形包含第 j 个点。如果一个点刚好在矩形的 边上 ，这个点也被视为被矩形包含。

    思路¶

    经典二维偏序对问题。对原数据与问题数据放在一起排序后，根据其中第一个坐标的逆序进行遍历，就可以保证第一个维度保持偏序，然后计算第二个维度的前缀和即可。事实上，我们可以将一维偏序问题，即逆序对问题中的下标代表着第一维偏序关系，元素值则代表第二维的偏序关系

    typedef pair<int,int> pii;
typedef pair<pii,int> piii;
vector<int> countRectangles(vector<vector<int>>& rectangles, vector<vector<int>>& points) {
    int INF = 1e9+1;
    vector<piii> vec;
    for(int i=0;i<rectangles.size();i++){
        vec.push_back(piii(pii(rectangles[i][0],rectangles[i][1]),INF));
        n = max(n,rectangles[i][1]);
    }
    for(int i=0;i<points.size();i++){
        vec.push_back(piii(pii(points[i][0],points[i][1]),i));
        n = max(n,points[i][1]);
    }
    n++;
    this->tree = vector<int>(n,0);
    sort(vec.begin(),vec.end());
    vector<int> ans(points.size());
    for(int i=vec.size()-1;i>=0;i--){
        if(vec[i].second==INF) add(vec[i].first.second,1);
        else ans[vec[i].second] = preSum(n-1)-preSum(vec[i].first.second-1);
    }
    return ans;
}
