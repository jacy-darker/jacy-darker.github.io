    二分查找¶

    基本思想¶

    使用\(O(log(n))\)的复杂度对有序列表进行查找，又称折半查找。根据二分点进行判断，二分点将整个待查找子区间分成了左右两部分，通过判断左右部分边界节点的特性来判断下一个查找子区间是左右还是右边，从而对另一区间进行舍弃。

    适用特征¶

    • 问题具有两段性：一段满足某个性质，另一段不满足某个性质
    • 可以通过画图分析数据的两段性
    • 查找的区间不一定是给的数组，也有可能是答案的区间或者是代表其他含义的区间

    类型¶

    • 最基本的二分查找：当 nums[mid] == target 时可以立即返回
    • 查找左边界的二分查找：当 nums[mid] == target 时不要立即返回，收紧右侧边界以锁定左侧边界
    • 查找右边界的二分查找：当 nums[mid] == target 时不要立即返回，收紧左侧边界以锁定右侧边界
    • 查到第一个大于等于，最后一个小于等于等常见类型，使用ans=mid可以较为有逻辑的处理这类问题

    重复出现的区间¶

    问题¶

    给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。

    如果数组中不存在目标值 target，返回 [-1, -1]。

    你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

    示例 1：

2

    示例 2：

    示例 3：

输出：[-1,-1]

    思路¶

    是找到第一个大于等于目标值的和第一个大于目标值的。与之前不同的是找到相等的之后不返回，继续找中间的值进行判断

    class Solution { 
public:
    int binarySearch(vector<int>& nums, int target, bool lower) {
        int left = 0, right = (int)nums.size() - 1, ans = (int)nums.size();
        while (left <= right) {
            int mid = (left + right) / 2;
            if (nums[mid] > target || (lower && nums[mid] >= target)) {
                right = mid - 1;
                //this is important
                ans = mid;
            } else {
                left = mid + 1;
            }
        }
        return ans;
    }

    vector<int> searchRange(vector<int>& nums, int target) {
        int leftIdx = binarySearch(nums, target, true);
        int rightIdx = binarySearch(nums, target, false) - 1;
        if (leftIdx <= rightIdx && rightIdx < nums.size() && nums[leftIdx] == target && nums[rightIdx] == target) {
            return vector<int>{leftIdx, rightIdx};
        } 
        return vector<int>{-1, -1};
    }
};

    山峰数组的峰值索引¶

    问题¶

    符合下列属性的数组 arr 称为 山脉数组 ：

    • arr.length >= 3
    • 存在i使得：
      • arr[0] < arr[1] < ... arr[i-1] < arr[i]
      • arr[i] > arr[i+1] > ... > arr[arr.length - 1]

    给你由整数组成的山脉数组 arr ，返回满足 arr[0] < arr[1] < ... arr[i - 1] < arr[i] > arr[i + 1] > ... > arr[arr.length - 1]的下标 i 。

    你必须设计并实现时间复杂度为 O(log(n)) 的解决方案。

    示例 1：

    1
2
    输入：arr = [0,1,0]
输出：1

    示例 2：

    1
2
    输入：arr = [0,2,1,0]
输出：1

    示例 3：

    1
2
    输入：arr = [0,10,5,2]
输出：1

    思路¶

    关键在于如何通过判断mid的特性来进行折半。一个包含峰值的区间的左节点一定是递增的，右节点一定是递减的，中间节点决定了mid成为下一个左节点或右节点才能保证这个性质。

    代码¶

    c++ int peakIndexInMountainArray(vector<int>& arr) { int n = arr.size(); int l = 1,r = n-2; int mid; while(l<=r){ mid = l + (r-l)/2; if(arr[mid-1] < arr[mid]) l = mid+1; else r = mid-1; } return r; }

    寻找旋转排序数组中的最小值 II¶

    问题¶

    已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转后，得到输入数组。例如，原数组 nums = [0,1,4,4,5,6,7] 在变化后可能得到：

    • 若旋转 4 次，则可以得到 [4,5,6,7,0,1,4]
    • 若旋转 7 次，则可以得到 [0,1,4,4,5,6,7]

    注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。

    给你一个可能存在 重复 元素值的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。

    你必须尽可能减少整个过程的操作步骤。

    示例 1：

    1
2
    输入：nums = [1,3,5]
输出：1

    示例 2：

    1
2
    输入：nums = [2,2,2,0,1]
输出：0

    提示：

    • n == nums.length
    • 1 <= n <= 5000
    • -5000 <= nums[i] <= 5000
    • nums 原来是一个升序排序的数组，并进行了 1 至 n 次旋转

    思路¶

    重点在于如何理解本题的二段性。与以往的二分不同，本题判断某一个区间的二段性是根据某个区间的右节点与nums[0]进行对比。

    int findMin(vector<int>& nums) {
    int n = nums.size();
    int l = 0 ,r = n-1;
    int mid;
    while(r>l&&nums[r]==nums[0]) r--;
    while(l<=r){
        mid = l + (r-l)/2;
        if(nums[mid] < nums[0]) r = mid-1;
        else l = mid+1;
    }
    l = mid-1;
    r = mid+1;
    if(l<0) l = n-1;
    if(r==n) r = 0;
    return min(nums[mid],min(nums[l],nums[r]));
}

    bool vaild(vector<int>& weights, int days,int c){
    int i = 0;
    int sum;
    int n = weights.size();
    while(days>0){
        sum = 0;
        while(i<n&&sum+weights[i]<=c) sum += weights[i++];
        if(sum==0&&i!=n) return false;
        days--;
    }
    return i==n;
}
int shipWithinDays(vector<int>& weights, int days) {
    int n = weights.size();
    long long l = (long long ) *max_element(weights.begin(),weights.end());
    long long r = (long long ) accumulate(weights.begin(),weights.end(),0);
    long long mid = r;
    int ans = r;
    while(l<=r){
        mid = l + (r-l)/2;
        if(vaild(weights,days,mid)) r = mid-1,ans = mid;
        else l = mid+1;
    }
    return ans;
}

    打家劫舍IV¶

    问题¶

    沿街有一排连续的房屋。每间房屋内都藏有一定的现金。现在有一位小偷计划从这些房屋中窃取现金。

    由于相邻的房屋装有相互连通的防盗系统，所以小偷 不会窃取相邻的房屋 。

    小偷的 窃取能力 定义为他在窃取过程中能从单间房屋中窃取的 最大金额 。

    给你一个整数数组 nums 表示每间房屋存放的现金金额。形式上，从左起第 i 间房屋中放有 nums[i] 美元。

    另给你一个整数 k ，表示窃贼将会窃取的 最少 房屋数。小偷总能窃取至少 k 间房屋。

    返回小偷的 最小 窃取能力。

    示例 1：

    1

    输入：nums = [2,3,5,9], k = 2
输出：5
解释：
小偷窃取至少 2 间房屋，共有 3 种方式：
- 窃取下标 0 和 2 处的房屋，窃取能力为 max(nums[0], nums[2]) = 5 。
- 窃取下标 0 和 3 处的房屋，窃取能力为 max(nums[0], nums[3]) = 9 。
- 窃取下标 1 和 3 处的房屋，窃取能力为 max(nums[1], nums[3]) = 9 。
因此，返回 min(5, 9, 9) = 5 。

    思路和前面几题的二分一致，但重点在于valid函数的构建。再该题中，是否打劫某一个下标，不会影响到后面的判断策略，可以画个例子判断一下。将Y表达为数值小于等于目标金额，N表示为大于不能打劫。对于YYNY这种情况，第一个打劫的对象选择第一个或者选择第二个，都不会影响最终的打劫数量，其他情况也是如此。

    int hIndex(vector<int>& citations) {
    int n = citations.size();
    int l = 0,r = n-1;
    int mid;
    while(l<=r){
        mid = l + (r-l)/2;
        if(citations[mid]>=n-mid) r = mid-1;
        else l = mid+1;
    }
    return n-l;
}

    到达终点数字¶

    问题¶

    在一根无限长的数轴上，你站在0的位置。终点在target的位置。

    你可以做一些数量的移动 numMoves :

    • 每次你可以选择向左或向右移动。
    • 第 i 次移动（从 i == 1 开始，到 i == numMoves ），在选择的方向上走 i 步。

    给定整数 target ，返回 到达目标所需的 最小 移动次数(即最小 numMoves ) 。

    示例 1:

    1
2
3
4
5
6
    输入: target = 2
输出: 3
解释:
第一次移动，从 0 到 1 。
第二次移动，从 1 到 -1 。
第三次移动，从 -1 到 2 。

    示例 2:

    1
2
3
4
5
    输入: target = 3
输出: 2
解释:
第一次移动，从 0 到 1 。
第二次移动，从 1 到 3 。

    提示:

    • -109 <= target <= 109
    • target != 0

    思路¶

    • 至少需要走\(min_k\{1+2+\cdots+k>= target \}\)步
    • 如果走了x步，将向左向右看成是1到x步连加时每个数字前面是+或者-。\(1+2+\cdots+x - target = diff\)，diff/2就是我们要从1,\cdots,x中找一个子集的和等于diff/2。因此，如果diff是奇数则不行，如果是偶数，可以证明一定可以找到这样一个子集

    int reachNumber(int target) {
    target = abs(target);
    int k = 0;
    while (target > 0) {
        k++;
        target -= k;
    }
    return target % 2 == 0 ? k : k + 1 + k % 2;
}

    收苹果¶

    问题¶

    给你一个用无限二维网格表示的花园，每一个 整数坐标处都有一棵苹果树。整数坐标 (i, j) 处的苹果树有 |i| + |j| 个苹果。

    你将会买下正中心坐标是 (0, 0) 的一块 正方形土地 ，且每条边都与两条坐标轴之一平行。

    给你一个整数 neededApples ，请你返回土地的 最小周长 ，使得 至少 有 neededApples 个苹果在土地 里面或者边缘上。

    |x| 的值定义为：

    • 如果 x >= 0 ，那么值为 x
    • 如果 x < 0 ，那么值为 -x

    思路¶

    推导数学公式，边长为n，周长为8n的苹果为2n(n+1)(2n+1)
