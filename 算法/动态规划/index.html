    动态规划¶

    基本思想¶

    将待求解问题分解成若干子问题，先求解子问题，再结合这些子问题得到原问题的解。同时，在求解过程中，保留已解决的子问题的解，在需要时再查找已求得的解，就可以避免大量重复计算。

    适用特征¶

    • 求解的问题是组合优化问题
    • 求解过程需要多步判断，从小到大依次求解
    • 子问题目标函数最优解之间存在依赖关系

    基本步骤¶

    1. 分析最优解结构
    2. 递归的定义最优值
    3. 以自底向上的方式计算最优值
    4. 根据计算最优值时得到的信息构造最优解

    要素¶

    • 最优子结构：当问题的最优解包含其子问题的最优解时，称该问题具有最优子结构性质。
    • 考察最优子结构时，先假设规模k的子问题已经达到了最优解
    • 接着在下一步做出一个选择，考虑如何达到k+1步的最优解，借此得到的策略就是状态转移方程

    • 证明一个问题是否具有最优子结构，需要用到反证法。如果一个问题的的最优解不包含子问题的最优解，那么用子问题的最优解替换之前的非最优解，如果总问题仍然最优，则说明有最优子结构

    • 子问题重叠：当用递归算法自顶向下求解此问题时，每次产生的子问题并不总是新问题，有些子问题被反复计算。

    • 备忘录(表格)：用表格保存已解决的子问题的答案，在下次需要求解子问题时，只要简单的查看该子问题的解，而不用重新计算。
    • 重构最优解：在需要知道最优解是如何得到的时候，我们可以将每次最优选择记录下来。

    优化¶

    • 贪心：
    • 滚动数组优化空间：如果二维或三维状态转移方程的某一维度只依赖于上一时刻，则可以优化这一空间。但需要注意的是，优化之后要特别注意遍历顺序，必须保证优化后的状态转移方程中所需要用到的数据是来自上一次的，即未被本轮优化过的。一般遍历顺序与其他维度的优化方程大小关系相反。

    类型¶

    • 背包问题
    • 区间DP：令状态\(dp[i][j]\)表示将下标位置i到j所有元素合并能获得的价值的最大值，那么dp[i][j] =max(dp[i][k]+dp[k+1][j]+cost)，cost为将这两组元素合并起来的代价。
    • 记忆化搜索
    • 状态压缩
    • 树形DP
    • DAG上的DP

    问题¶

    给定\(n\)个矩阵\(\{A_1,A_2,...,A_n\}\),其中\(A_i\)和\(A_{i+1}\)是可乘的，确定矩阵乘法顺序，使得元素相乘的次数最少

    void MatrixChain(int *p,int n,int **m,int **s){//p是输入参数-代表每个矩阵的行列数，m为最优值数组，s为记录最优断开位置的数组
    for(int i = 1;i<=n;i++)m[i][i] = 0;//初始化
    for(int r = 2;r<=n;r++){//斜线递归次数
        for(int i=1;i<=n-r+1;i++){
            int j = i+r-1;
            m[i][j] = m[i+1][j] + p[i-1]*p[i]*p[j];//作为第一次的初始值，作为后面的参考，如有更优值则特换
            s[i][j] = i;        //在i处断开
            for(int k=i+1;k<j;k++){//判断是否有更优值
                int t = m[i][k]+m[k+1][j]+p[i-1]*p[k]*p[j];
                if(t<m[i][j]){//找到更优解
                    m[i][j] = t;
                    s[i][j] = k;
                }
            }
        }
    }
}

    //c[i][j]存储Xi和Yi的最长公共子序列的长度，

    电路布线¶

    问题¶

    在一块电路板的上、下两端分别有\(n\)个接线柱。根据电路设计，要求用导线\((i,\pi(i))\)将上段接线柱与下端接线柱相连，其中\(\pi(i)\)是\(\{1,2,...,n\}\)的一个排列。导线\((i,\pi(i))\)称为该电路板上的第\(i\)条连线。对于任何\(i<j\),第\(i\)条连线和第\(j\)条连线相交的充要条件是\(\pi(i)>\pi(j)\).

    电路布线问题要确定将哪些连线安排在第一层上，使得该层上有尽可能多的连线。或换句话说，该问题要求确定导线集\(Nets=\{i,\pi(i),1<=i<=n\}\)的最大不相交子集

    思路¶

    与最长公共子序列同构

    记\(N(i,j)=\{t|(t,\pi(t))\in Nets,t<=i,\pi(t)<=j\},N(i,j)\)的最大不相交子集为\(MNS(i,j)\).\(Size(i,j)=|MNS(i,j)|\)

    \(i=1\)

    \[ Size[1][j]= \begin{cases} 0 & j<\pi(1)\\ 1 & j>=\pi(1) \end{cases} \]

    \(i>1\)

    \[ Size[i][j]= \begin{cases} Size[i-1][j] & j<\pi(i)\\ max\{Size[i-1][j],Size[i-1][\pi(i)-1]+1\} & j>=\pi(i) \end{cases} \]

    void MNS(int c[],int n,int **size){
    int i,j;
    for(j=1;j<=n;j++){//i=1
        if(j<c[1])size[1][j] = 0;
        else size[1][j] = 1;
    }
    for(i=2;i<n;i++){//i>1
        for(j=1;j<=n;j++){
            if(j<c[i])size[i][j] = size[i-1][j];
            else{
                if(size[i-1][j]>size[i-1][c[i]-1]+1)size[i][j] = size[i-1][j];
                else size[i][j] = size[i-1][c[i]-1]+1;
            }
        }
    }
}

    void Knapsack(int* v,int* w,int c,int n,int** m){
    for(int j=0;j<=c;j++){//先确立边界值
        if(j<w[n])m[n][j] = 0;
        else m[n][j] = v[n];
    }
    for(int i=n-1;i>=1;i--){
        for(int j=0;j<=c;j++){
            if(j<w[i])m[i][j] = m[i+1][j];
            else m[i][j] = max(m[i+1][j],m[i+1][j-w[i]]+v[i]);
        }
    }
}

void Traceback(int** m,int* w,int c,int n,int* x){
    for(int i=1;i<n;i++){

            x[i] = 1;

    可以使用滚动数组

    变种¶

    • 完全背包：所有物体都可以取无限次
    \[ f_{i, j}=\max _{k=0}^{+\infty}\left(f_{i-1, j-k \times w_i}+v_i \times k\right) \]

    优化后

    \[ f_{i, j}=\max \left(f_{i-1, j}, f_{i, j-w_i}+v_i\right) \]
    • 多重背包：每个物品有\(k\)次
    \[ f_{i, j}=\max _{k=0}^{k_i}\left(f_{i-1, j-k \times w_i}+v_i \times k\right) \]

    优化：将每一个物品的k次按二进制拆分成组，分别可以取1,2,4,..次。这样如果最优解选择了4次。只需要选择分组为4的物品即可。

    • 混合背包：有的取1次，有的取k次，有的无限取

    遍历物品，如果是0-1背包，使用0-1背包的状态转移方程；如果是完全背包，则采用完全背包的...

    • 二维费用背包：背包受到容量和花费的限制

    使用滚动数组优化即可

    • 分组背包：物品属于不同的组别，一组只能取一个物品。

    遍历组，使用0-1背包公式。注意遍历顺序，维度顺序是组->背包容量->该组物品

    • 有依赖背包：物品有主和附之分。只能选择主或者主+附进行购买。每种策略只能选一种，所以可以当做分组背包处理
    • 泛化物品的背包：这种背包，没有固定的费用和价值，它的价值是随着分配给它的费用而定。在背包容量为 的背包问题中，当分配给它的费用为\(v_i\)时，能得到的价值就是\(f(v_i)\)。这时，将固定的价值换成函数的引用即可。
    • 其他
    • 最优解的策略：使用数组记录状态转移时候的选择

    }

        for (int j = 1; j < n; j++) {
            if (grid[i][j] >= 0) {
                maxgt[i][j] = max(maxgt[i][j - 1], maxgt[i - 1][j]) * grid[i][j];
                minlt[i][j] = min(minlt[i][j - 1], minlt[i - 1][j]) * grid[i][j];

                maxgt[i][j] = min(minlt[i][j - 1], minlt[i - 1][j]) * grid[i][j];
                minlt[i][j] = max(maxgt[i][j - 1], maxgt[i - 1][j]) * grid[i][j];
            }
        }
    }
    if (maxgt[m - 1][n - 1] < 0) {
        return -1;
    } else {
        return maxgt[m - 1][n - 1] % mod;

    思路¶

    设\(f[i][j]\)为安排i天的工作，最后一天工作为j的最小复杂度。 $$ f[i][j] = min_{i-1<=k<j}(f[i-1][k]+min_{k+1=<t<=j}(a[t])) $$ 简单的动态规划的复杂度是\(O(dn^2)\)。我们可以使用单调栈来进行优化，使得时间复杂度降到\(O(dn)\)。

    考虑第\(i\)天，对于元素\(j\),\(left[j]\)是左边第一个大于当前元素的下标。如果\(i\)天的工作选择\(left[j]+1 <=k<= j\)，则第\(i\)天的复杂度为\(a[j]\)，总复杂度为\(f[i-1][k]+a[j]\)。如果第\(i\)天工作有\(left[j]\)，则总复杂度为\(f[i][left[j]]\)。取最小值即可。

    int minDifficulty(vector<int> &a, int d) {
    int n = a.size();
    if (n < d) return -1;

    int f[d][n];
    f[0][0] = a[0];
    for (int i = 1; i < n; i++)
        f[0][i] = max(f[0][i - 1], a[i]);
    for (int i = 1; i < d; i++) {
        stack<pair<int, int>> st; // (下标 j，从 f[i-1][left[j]] 到 f[i-1][j-1] 的最小值)
        for (int j = i; j < n; j++) {
            int mn = f[i - 1][j - 1]; // 只有 a[j] 一项工作
            while (!st.empty() && a[st.top().first] <= a[j]) { // 向左一直计算到 left[j]
                mn = min(mn, st.top().second);
                st.pop();
            }
            f[i][j] = mn + a[j]; // 从 a[left[j]+1] 到 a[j] 的最大值是 a[j]
            if (!st.empty()) // 如果这一段包含 <=left[j] 的工作，那么这一段的最大值必然不是 a[j]
                f[i][j] = min(f[i][j], f[i][st.top().first]); // 答案和 f[i][left[j]] 是一样的
            st.emplace(j, mn); // 注意这里保存的不是 f[i][j]
        }
    }
    return f[d - 1][n - 1];
}

int maxHeight(vector<vector<int>>& cuboids) {

    sort(cuboids.begin(),cuboids.end(),[](const auto &u,const auto &v){
        if(u[0]==v[0]){
            if(u[1]==v[1]){
                return u[2]<v[2];
            }
            else return u[1]<v[1];
        }
        else return u[0]<v[0];
    });

    vector<int> dp(n,1);
    for(int i=0;i<n;i++){
        dp[i] = cuboids[i][2];
        for(int j=i-1;j>=0;j--){
            if(check(cuboids[j],cuboids[i])){
                dp[i] = max(dp[i],cuboids[i][2]+dp[j]);
            }
        }
    }
    return *max_element(dp.begin(),dp.end());
}

    int n = s.size();
    vector<int> pre(n,0);

        long long time = t[0];

    等差数列划分¶

    问题¶

    给你一个整数数组 nums ，返回 nums 中所有 等差子序列 的数目。

    int numberOfArithmeticSlices(vector<int>& nums) {
    int n = nums.size();
    vector<unordered_map<long long,int>> dp(n);
    for(int i=1;i<n;i++){
        for(int j=0;j<i;j++){
            long long d = (long long)nums[i] - (long long)nums[j];
            dp[i][d] += dp[j][d]+1;
